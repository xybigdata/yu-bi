## 6. 当前有效结论

### 6.1 前端依赖侧

- `react-dev-inspector` 已完成移除，不再作为后续事项
- `react-window` 仍是运行时依赖，但使用面较窄，适合作为后续单独审计专题
- `flexlayout-react`、`react-grid-layout` 版本偏旧，但仍在关键交互主链，不适合无证据替换
- `react-quill 2` 当前先以稳定化小修为主，不进入重写或替代

### 6.2 时间体系

- 主方向已明确：继续收口到 `formatDatartDate(...)` 与 `toDatartDayjs(...)`
- 重点不是升级 `dayjs` 本身，而是消化分散的旧调用链
- 看板区间结束值调整与图表预览区间时间筛选器，已开始收口到统一日期解析入口
- 图表数据请求中的旧 `formatTime(...)` 别名链已开始退出
- 一批区间时间组件的二元时间值转换，已开始收口到统一日期工具
- 每一批都应控制在小范围内，并执行完整前端门禁

### 6.3 后端高风险链路

- `Shiro` 仍承载认证与授权主链，现阶段只做观察，不做迁移
- `Calcite` 仍深度参与 SQL 解析、变量替换、方言处理，现阶段不做升级或替代
- 后端现代化当前更适合做“兼容验证”和“小范围依赖健康维护”，不适合结构性改写
